# 資料夾結構同步

用 `core-structure-sync` 檢查 Life.os 的實際資料夾結構有沒有跟 `README.md`／`CLAUDE.md` 記載的脫節，如果有落差，會先問你原因再更新文件——不會自己腦補原因就動手改。

## 什麼時候用

- 手動輸入 `/core-structure-sync` 做一次完整檢查。
- 任何任務過程中，只要新增/刪除/搬移/改名了資料夾，就該主動跑一次，而不是默默留著沒人知道。`core-init-latex`、`core-latex-tidy` 等其他 skill 執行完如果動到資料夾結構，也會提醒可以跑這個。

## 流程

1. 列出目前實際的資料夾結構（第一層 + 有意義的第二層）。
2. 讀 `CLAUDE.md` 的「資料夾結構」章節、`README.md` 的場景清單，抓出文件裡記錄的結構。
3. 比對三種差異：新增、消失/改名、內容漂移（資料夾還在但用途已經不一樣）。
4. 如果沒有差異：直接回報「一致，不需要更新」，結束。
5. **如果有差異，一定會先問你**：這是新增了什麼功能？還是純粹改名/搬移？新功能的話簡短說一下做什麼用的。不會自己猜原因就動手改文件。
6. 依回答同步更新 `CLAUDE.md`（資料夾結構章節）跟 `README.md`（依場景查清單，必要時也更新「幾條線」章節）——兩份文件只改一份是不允許的。

## 相關檔案位置

- skill 定義：`.claude/skills/core-structure-sync/`
- 會被更新的文件：根目錄 [README.md](../README.md)、[CLAUDE.md](../CLAUDE.md)

## 注意事項

- 這是會被重複執行的檢查，每次都會重新掃描實際結構，不依賴上次跑的結果。
- 回報只會用 2-3 句話說明變動跟改了哪裡，不會整段貼文件 diff。
