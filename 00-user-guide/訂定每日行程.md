# 每日行程同步 SOP

## 該用哪個指令？

三選一，依情況挑：

| 情況 | 用哪個 | 為什麼 |
|---|---|---|
| 雲端跑過，要再加到 Note、Reminder | `/core-reminders-sync` | 雲端算好，補 Reminders/Notes |
| 雲端算過，但加新的行程 | `/core-daily-sync` | 重掃 `schedule/`，重新安排 |
| 雲端沒成功 | `/core-daily-sync` | 順便把雲端漏掉的 Calendar 補上 |

不確定就用 `/core-daily-sync`，一定不會漏掉東西

## 三個入口分別在做什麼

### `/core-cloud-sync`（雲端，全自動，平常不用管）

- 台灣時間 06:00 由 GitHub Actions 自動觸發；也可以在手機上手動觸發（GitHub App → Actions → core-cloud-sync → Run workflow），方便隨時想立刻同步時用。
- 依序執行：`sub-daily-check`（整理待辦）→ `sub-caldav-sync`（透過 CalDAV 直連 iCloud，把有明確時間的任務寫進 Calendar「daily」行事曆）→ `sub-notion-calendar-sync`（把當天待辦單向鏡像進 Notion「Daily-Sync」資料庫）。
- **Calendar + Notion 鏡像**。Reminders/Notes 沒有可用的雲端 API（Apple 這幾年把 Reminders 換成私有同步協定），這兩項只能靠本機。Notion 那段的細節（標籤欄位規則、一次回補歷史等）見 [00-user-guide/Notion同步.md](Notion同步.md)。

### `/core-daily-sync`（本機，完整版）

- 依序執行：`sub-daily-check`（重新掃描 `schedule/`，整理今天/明天/未來 7 天待辦）→ `sub-apple-sync`（同步 Calendar + Reminders + Notes 三個 App）。
- 重新分析一次 `schedule/`，所以能抓到雲端跑完之後你才加進去的新資料。
- Calendar 那段會自動檢查雲端是否已經同步過同一筆事件，不會重複建立（兩條路徑寫進同一個「daily」行事曆，互相看對方的狀態檔）。

### `/core-reminders-sync`（本機，輕量版）

- 不重新分析，直接 `git pull` 抓雲端已經算好的 `tasks.json`。
- 只同步 Reminders + Notes，完全不碰 Calendar（信任雲端已經處理過）。
- 如果今天的 `tasks.json` 還不存在（例如雲端今天沒跑），會提醒你先跑 `/core-daily-sync` 或去手機觸發雲端，不會自己臨時湊資料。

## 執行後要檢查什麼

- **Calendar.app**：確認事件都在「daily」行事曆裡
- **Reminders.app**：確認待辦都在「daily-sync」清單裡
- **Notes.app**：確認備忘錄在「daily-sync」資料夾裡
- **`schedule/schedule-log.md`**：如果 `sub-daily-check` 有搬移過期檔案，這裡會留紀錄（搬去 `trash-can/` 的原因）。建議偶爾翻一下，確認沒有搬錯東西。

## 固定的 Apple 端容器名稱

- Calendar：iCloud 帳號底下的「**daily**」行事曆——這個要先手動建立一次（Calendar.app > File > New Calendar，帳號選 iCloud，命名為 daily），程式不會、也不該自動建立（AppleScript 沒辦法指定帳號，自動建的話會建在本機、不會同步到 iPhone）。
- Reminders：iCloud 帳號底下的「**daily-sync**」清單，所有提醒都進同一個清單（不是每天開新清單），完成的項目會標記完成、不刪除，想清乾淨就用 Reminders.app 內建的「已完成・清除」。
- Notes：iCloud 帳號的「**daily-sync**」資料夾，一天一則筆記，已存在就不會自動覆寫。

## 當天待辦的檔案長什麼樣

`02-zettelkasten/03-Calendar/<今天>/` 底下：

- `<今天>.md` — 「大表」，一張表格（時間／任務／分類／卡片連結），不是完整待辦內容。
- `tasks/` — 今天每一筆任務各自一張 `.md` 卡片（含 frontmatter：id/title/date/time/type/source_file），打勾的 checkbox 在卡片裡。
- `tasks.json` — 機器讀格式，給 `sub-apple-sync`／`sub-caldav-sync`／`sub-notion-calendar-sync` 用，含今天／明天／未來 7 天，不受上面兩者影響。

這個「一任務一卡片＋大表索引」格式預設只套用到改版後新產生的日期；改版前的舊資料夾預設維持原本純 checkbox 條列格式，不會主動回溯轉換。如果想把某個舊資料夾也轉成新格式，直接跟我說要轉哪一天即可，我會依同樣的規則手動處理（`2026-07-20/` 已經這樣轉過）。這個格式已經實測跟 `sub-notion-calendar-sync` 相容，表格＋卡片連結會正常同步進 Notion。

## 注意事項

- Notes.app 沒辦法用程式建立真正可勾選的 checklist（Apple 沒開放這個 API），Reminders.app 才是可勾選待辦的正解；Notes 比較適合當「當天完整資訊的可讀筆記」。
- 第一次在本機執行 `sub-apple-sync` 時，macOS 可能跳出「是否允許自動化控制 Calendar / Reminders / Notes」的系統對話框，需要手動允許一次；如果因權限被拒報錯，去「系統設定 > 隱私權與安全性 > 自動化」開啟權限再重跑。
