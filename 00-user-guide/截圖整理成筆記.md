# 截圖整理成筆記

把看教學影片截的一堆圖，搭配重點筆記，整理成一份圖文並茂的使用說明／學習筆記。

## 該用哪個？

| 情況 | 用哪個 |
|---|---|
| 截圖還沒改檔名、也還沒寫筆記，想一次做完 | `/core-screenshot-notes` |
| 只想改檔名，筆記晚點再寫 | `/sub-screenshot-rename` |
| 檔名已經是 `NN_主題-重點.png` 格式，只想寫筆記 | `/sub-screenshot-writer` |

不確定就用 `/core-screenshot-notes`，它會依序呼叫另外兩個 skill。

## `/core-screenshot-notes` 整體流程

1. 先問「截圖都截好、放在同一個資料夾了嗎？」——還沒截好會停在這裡，等你截完再重新叫用。
2. 確認截圖資料夾路徑、跟你想傳達的重點（對話裡講過就不會重問）。
3. 呼叫 `sub-screenshot-rename` 把檔名改成看得懂的格式。
4. 呼叫 `sub-screenshot-writer` 產出 `使用說明.md`。
5. 回報：處理了幾張截圖、檔名怎麼改的、筆記檔案在哪裡。

## 檔名格式

```
NN_主題-重點.png
```

- `NN`：兩位數流水號，維持截圖順序；資料夾裡已經有同格式舊檔名的話會接續最大值往後排，不會跟舊的重複。
- `主題`：這組截圖的分類，例如「流程圖」「心智圖」。
- `重點`：這張圖要傳達的核心概念，精簡到看檔名就知道在講什麼。

改名前會先列出「原檔名 → 新檔名」對照給你看一眼，除非你已經說「不用確認直接做」。

## 筆記格式

產出（或附加到既有）`使用說明.md`，依主題分節，每個重點一個項目，圖片直接嵌入（不是超連結）：

```markdown
- **重點標題** — 說明文字。
  ![NN_主題-重點.png](NN_主題-重點.png)
```

如果資料夾裡已經有處理過的 `使用說明.md`，新一批截圖會用附加方式接續寫下去，不會整篇覆寫。
