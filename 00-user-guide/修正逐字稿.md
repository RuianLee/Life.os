# 修正逐字稿

用 `core-transcript-fix` 修正語音辨識產生的逐字稿裡的辨識錯誤（同音錯字、斷句錯誤、專有名詞誤植），不新增也不刪減內容。

## 怎麼用

- 定位到要修正的逐字稿檔案（IDE 目前開啟/聚焦的檔案，或訊息裡明確指定路徑），輸入 `/core-transcript-fix` 即可，不用額外說明。
- 只會處理明顯是語音辨識輸出的段落（大段沒標點、明顯同音錯字、語意不連貫）；檔案裡已經是手打的正確文字（手寫綱要、標題、引用）不會被動。

## 修正原則

- 同音字／斷句錯誤：改回正確用字，補上合理標點。
- 領域慣用術語：先判斷這份逐字稿在講什麼領域（信仰、技術、課程……），再依領域修正專有名詞。
- 人名：只改能合理確認的，不確定的保留最接近的音譯猜測，不會臆造成別的名字。
- 可查證的事實／引用（經節編號、年份、書名等）：跟事實對不上的地方會依正確內容修正。
- 低信心片段：辨識結果嚴重破碎、猜不出原意的地方，會盡量重建並標記 `[?]`，不會平白刪掉。

## 執行後要檢查什麼

- 回報裡會列出修正了哪幾段、標了幾處 `[?]`。
- **`[?]` 標記的地方不保證正確**，如果需要 100% 準確（正式引用、存查用途），建議回去對照原始錄音確認。
