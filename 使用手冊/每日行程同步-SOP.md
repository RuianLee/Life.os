# 每日行程同步 SOP

適用範圍：`行程/` 資料夾的每日待辦整理，以及同步到 Calendar.app / Reminders.app / Notes.app（都會經 iCloud 同步到 iPhone/iPad）。

**目前是純手動模式，沒有排程**——需要你自己在 Claude Code 裡輸入指令才會執行。

## 日常怎麼用

每天想整理待辦、同步到手機時，在這個專案目錄下開 Claude Code，輸入：

```
/daily-sync
```

這會依序做三件事：

1. **`/daily-check`**：掃描 `行程/` 底下所有檔案（含截圖/圖片），整理出「今天／明天／未來 7 天」的待辦，寫成：
   - `每日整理/<今天日期>/<今天日期>.md`（人看的整理，含 checkbox）
   - `每日整理/<今天日期>/tasks.json`（給下面兩步用的結構化資料）
   - 同時檢查 `行程/` 裡有沒有已經過期、沒參考價值的檔案，保守地搬到根目錄 `垃圾桶/`，並把判斷結果寫進 `行程/schedule-log.md`
2. **`/ios-sync`**：把 `tasks.json` 同步到 Calendar.app 的「Life.os」行事曆（有明確時間的事項）與 Reminders.app 的「Life.os」清單（只有日期的待辦）。重跑不會建立重複事件。
3. **`/notes-sync`**：把當天的待辦整理成 Notes.app「Life.os」資料夾裡的一則筆記，方便只帶平板時查看。

如果只想跑其中一步，也可以單獨輸入 `/daily-check`、`/ios-sync`、`/notes-sync`。

## 執行後要檢查什麼

- **Calendar.app**：打開「Life.os」行事曆，確認今天的時間點事件都在（例如報到、繳費時間）。
- **Reminders.app**：打開「Life.os」清單，確認待辦都在，可以直接點掉打勾（這是真正可勾選的清單）。
- **Notes.app**：打開「Life.os」資料夾，會有一則「`<日期> 待辦`」的筆記。
  - ⚠️ **這則筆記預設不是可勾選的 checklist**，只是條列文字（技術原因見下方「已知限制」）。
  - 如果想要能打勾：在 iPad 或 Mac 上打開這則筆記，全選條列的文字，點工具列的「打勾清單」格式化按鈕，就會變成真正可以點的 checklist。
  - **同一天的筆記只會建立一次，之後重跑 `/daily-sync` 不會覆寫它**，所以你手動轉換、勾選的進度是安全的，不用擔心被洗掉。
- **`行程/schedule-log.md`**：如果 `/daily-check` 有搬移過期檔案，這裡會留紀錄（搬去 `垃圾桶/` 的原因）。建議偶爾翻一下，確認沒有搬錯東西。

## 常見狀況

| 狀況 | 處理方式 |
|---|---|
| Calendar/Reminders/Notes 第一次同步時跳出「是否允許自動化控制」的系統對話框 | 手動允許即可，之後不會再問 |
| 想重新整理某一天的待辦（例如來源資料改了想強制重跑） | 直接刪掉 `每日整理/<日期>/` 資料夾再跑 `/daily-check`，會重新產生 |
| Reminders/Notes 裡多了重複項目 | 通常是待辦標題文字被重新措辭（daily-check 每次生成用詞可能略有不同）導致被判斷成「新任務」，舊的會在下次同步時自動清掉（Reminders 標記完成、Calendar 事件刪除），Notes 不會自動清（見下方限制） |
| 想整個重置、砍掉所有同步資料重來 | 1) 到 Calendar.app 刪除「Life.os」行事曆 2) 到 Reminders.app 刪除「Life.os」清單 3) 到 Notes.app 刪除「Life.os」資料夾 4) 刪除 `行程/.ios-sync-state.json` 與 `行程/.notes-sync-state.json`（這兩個檔案是本機快取，不會進 git）5) 重跑 `/daily-sync` |
| 想找回被搬到 `垃圾桶/` 的檔案 | 直接去 `垃圾桶/` 底下找，路徑結構跟原本 `行程/` 裡一樣，手動搬回去即可（沒有自動清空機制） |

## 已知限制

- **Notes.app 沒有辦法用程式建立真正可勾選的 checklist**：Notes 的 AppleScript 字典裡沒有 checklist 相關屬性，網路上流傳的 `<ul class="checklist">` HTML 技巧實測會被 Notes 正規化成普通條列，勾選狀態不會保留。真正想要「一開始就能打勾」的體驗，請用 Reminders.app（已經同步好，本來就是可勾選的清單）。
- **待辦的重複判斷是用「標題文字」比對的**：如果 `/daily-check` 兩次生成的待辦標題措辭不完全一樣（例如用詞順序微調），會被當成不同任務，導致舊的被清掉、新的被建立，而不是「更新」。這在實務上影響不大（同一天內容通常穩定），但如果你發現同一件事一直重複新增/清除，這就是原因。
- **目前完全手動、沒有排程**：不會自動在背景執行。如果之後想改回排程自動跑，技術上是可行的（之前有做過、也驗證過核心邏輯沒問題），但會需要另外處理 macOS 的權限限制（launchd 背景程式讀取 `~/Desktop` 底下的檔案會被系統擋下來，需要手動到「系統設定 > 隱私權與安全性 > 完整磁碟取用權限」加入 `/bin/zsh`）。細節見 `小專案/每日同步技術文件/`。
