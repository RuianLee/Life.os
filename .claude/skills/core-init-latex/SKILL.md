你現在要建立一份「LaTeX 格式範例」的 `.tex` 檔案，這份檔案本身要能直接用 xelatex 編譯出一份「每種格式長什麼樣子」的對照 PDF，同時原始碼裡每個範例也要看得懂語法怎麼寫，方便使用者以後想用某個格式時直接來這裡複製貼上。

## 0. 確認要建在哪裡

- 每次執行都要問使用者：要建在哪個資料夾、檔名是什麼（預設可以建議 `LaTeX格式範例.tex`）。不要自己假設路徑，因為這個檔案放的位置沒有固定慣例。
- 如果目標路徑已經有同名檔案，跟使用者確認要覆蓋還是換個檔名，不要沒問就蓋掉。

## 1. 找專案既有的 .tex 慣例當基礎

- 在專案裡找一個既有的、已知能編譯成功的 `.tex` 檔案（例如 `召會生活/聖經陪讀/製作自己的聖經陪讀教材/` 底下的檔案）當參考，抓它的前導碼（`\documentclass`、字型設定 `\usepackage{fontspec}` + `\usepackage{xeCJK}` + `\setCJKmainfont` 等、`\usepackage{geometry}`、`\usepackage{enumitem}` 等）。
- 用同一套前導碼組出範例檔案的開頭，確保跟專案裡其他 `.tex` 一樣可以用 `xelatex` 直接編譯、中文字型正常顯示，不要另外發明一套沒驗證過的前導碼。
- 如果專案裡真的找不到任何既有 `.tex` 可以參考，才退回用 CLAUDE.md 記載的慣例（`ctex` + `xelatex`）當前導碼。

## 2. 內容涵蓋的格式範例

每個格式前面用 `\section*{}` 或註解清楚標出格式名稱（方便使用者搜尋/瀏覽原始碼），內容至少涵蓋：

- 文件標題：`\title{}` `\author{}` `\date{}` + `\maketitle`
- 大標題：`\section{}` 與不編號版 `\section*{}`
- 中標題：`\subsection{}` 與不編號版 `\subsection*{}`
- 一般段落文字
- 編號清單：`enumerate`（含 `leftmargin=*, label=\arabic*.` 這種自訂編號寫法）
- 條列清單：`itemize`
- 粗體 `\textbf{}`、斜體 `\textit{}`、底線 `\underline{}`
- 引用區塊：`quote` 環境
- 強調換行與分隔線：`\bigskip` + `\hrule`
- 註腳：`\footnote{}`
- 表格：基本 `tabular` 範例
- 每個範例旁邊用該格式本身示範一小段有意義的中文範例文字（不用是無意義的 lorem ipsum），讓編譯出來的 PDF 看得出實際效果

## 3. 建立檔案

用 Write 把組好的內容寫到第 0 步確認的路徑。

## 4. 回報

跟使用者說檔案建在哪裡、涵蓋了哪些格式，並提醒可以直接 `xelatex` 編譯這份檔案看每種格式實際呈現的樣子；編譯完如果想整理產生的中間檔，可以用 `core-latex-tidy`。
