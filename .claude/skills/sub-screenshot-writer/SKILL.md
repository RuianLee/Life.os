你現在要把使用者一批已經改好檔名的截圖，搭配筆記重點，整理成一份圖文並茂的 Markdown 筆記。照下面步驟走。

## 0. 是否已經拿到資料夾與筆記重點

如果呼叫這個 skill 的訊息裡已經明確給了「截圖資料夾路徑」「檔名清單」與「筆記重點」（例如來自 `core-screenshot-notes` 總 skill 依序呼叫，接在 `sub-screenshot-rename` 之後），直接跳到「2. 檢查檔名格式」，不用再問使用者一次。

否則繼續下面第 1 步。

## 1. 確認截圖資料夾與筆記重點

- 確認截圖放在哪個資料夾。如果使用者訊息或目前開啟的檔案已經看得出資料夾路徑，直接用，不用重複問。
- 確認使用者的筆記重點。如果對話裡已經有筆記內容就直接用；如果完全沒提，才問使用者「想請你講一下這幾張圖大致是在講什麼、重點是什麼」。

## 2. 檢查檔名格式

看資料夾裡的圖片檔名是不是已經是 `NN_主題-重點.png` 格式：

- 如果已經是：直接進下一步。
- 如果還不是（例如還是螢幕截圖預設檔名）：告訴使用者這批截圖還沒改名，建議先跑 `sub-screenshot-rename` skill 把檔名改好再回來寫筆記；除非使用者明確說「不用改名，直接用原檔名寫筆記」，才照原檔名繼續。

## 3. 產出使用說明／學習筆記

在同一個資料夾建立（或更新）一份 `使用說明.md`（或使用者指定的檔名）：

- 依主題分節（`##`），每個重點一個項目：**粗體重點標題** — 說明文字（結合使用者筆記與圖片內容），下一行用圖片語法嵌入對應截圖：

  ```markdown
  - **重點標題** — 說明文字。
    ![NN_主題-重點.png](NN_主題-重點.png)
  ```

  **一定要用 `![]()` 讓圖片直接顯示，不要用 `[]()` 超連結格式。**

- 如果內容適合用小結表格、條列步驟等輔助整理（參考 `生產力工具/Ｗhimsical/使用說明.md` 的「小結：畫流程圖的形狀慣例」），可以加，但不要為了加而加。

- 如果資料夾裡已經有一份 `使用說明.md`，且裡面的圖片已經是 `NN_主題-重點` 格式（代表是之前處理過的），這次用附加（append）方式接續寫下去，不要整篇覆寫。

## 4. 回報

用幾句話說明：這次寫了哪些主題／重點、產出的筆記檔案路徑在哪。不用整篇貼出筆記內容。
