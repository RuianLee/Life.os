name: sub-screenshot-rename
description: 把某個資料夾裡一批教學截圖，依照畫面內容與使用者提供的重點，改成看得懂的檔名（NN_主題-重點.png 格式）。適用場景：使用者說「幫我把這些截圖改名」「這資料夾的截圖檔名太亂看不懂」之類的需求；也是 core-screenshot-notes 總流程的第一步，會被它依序呼叫。
---

你現在要把使用者截的一批圖片，依照畫面內容改成看得懂的檔名。照下面步驟走。

## 0. 是否已經拿到資料夾與筆記重點

如果呼叫這個 skill 的訊息裡已經明確給了「截圖資料夾路徑」與「筆記重點」（例如來自 `core-screenshot-notes` 總 skill 依序呼叫），直接跳到「3. 逐張查看截圖內容」，不用再問使用者一次。

否則繼續下面 1、2 步驟。

## 1. 確認截圖已經截好（不可跳過）

先問使用者：「這次要改名的截圖都已經截好、放在同一個資料夾了嗎？」

- 如果還沒截好：請使用者先去截圖，截完再重新叫用這個 skill，本次先停在這裡，不要往下做。
- 如果已經截好：才繼續下一步。

## 2. 確認截圖資料夾與筆記重點

- 確認截圖放在哪個資料夾。如果使用者訊息或目前開啟的檔案已經看得出資料夾路徑，直接用，不用重複問。
- 確認使用者的筆記重點。使用者可能已經在對話中打字講了每張圖（或每個段落）在講什麼，也可能還沒講——如果對話裡已經有筆記內容就直接用；如果完全沒提，才問使用者「想請你講一下這幾張圖大致是在講什麼、重點是什麼」（這批重點之後寫筆記時也用得到，記得保留給後續步驟或後續 skill 使用）。

## 3. 逐張查看截圖內容

用 Read tool 開啟資料夾裡的每一張截圖（圖片可以直接讀取查看），對照使用者的筆記重點，理解每張圖實際在畫面上呈現什麼、想傳達的重點是什麼。

## 4. 依內容把檔名改好

命名慣例（照 `生產力工具/Ｗhimsical/使用說明.md` 這組截圖的做法）：

```
NN_主題-重點.png
```

- `NN`：兩位數流水號，維持原本的截圖順序（用檔案的建立/修改時間排序，或使用者講的順序）。如果資料夾裡已經有一份 `使用說明.md`，且既有圖片已經是這個格式，流水號要接續既有最大值往後排，不要跟舊的重複、也不要重新改舊檔名。
- `主題`：這組截圖的分類，例如「流程圖」「心智圖」。
- `重點`：這張圖要傳達的核心概念，精簡到「看檔名就知道這張圖在講什麼」，不用完整句子。

用 `mv` 依序把原始檔名（例如螢幕截圖預設的檔名）改成新檔名。改名前列出「原檔名 → 新檔名」的對照給使用者看一眼再動手，除非使用者已經明確說「不用確認直接做」。

## 5. 回報

用幾句話說明：這次處理了幾張截圖、檔名怎麼改的（簡短列 before → after 或直接說「見上方對照表」）。如果是被 `core-screenshot-notes` 總 skill 呼叫，把改好的資料夾路徑、檔名清單、使用者給的筆記重點交給下一步（`sub-screenshot-writer`），不用另外印出來給使用者看。
