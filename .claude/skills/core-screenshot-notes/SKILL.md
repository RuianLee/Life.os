你現在要把使用者截的一批圖片，搭配筆記重點，整理成一份圖文並茂的 Markdown 筆記，並把圖片改成看得懂的檔名。這個 skill 本身不做細節，而是依序呼叫兩個子 skill 完成：`sub-screenshot-rename`（改檔名）→ `sub-screenshot-writer`（寫筆記）。照下面步驟走。

## 1. 確認截圖已經截好（不可跳過）

先問使用者：「這次要用的截圖都已經截好、放在同一個資料夾了嗎？」

- 如果還沒截好：請使用者先去截圖，截完再重新叫用這個 skill，本次先停在這裡，不要往下做。
- 如果已經截好：才繼續下一步。

## 2. 確認截圖資料夾與筆記重點

- 確認截圖放在哪個資料夾。如果使用者訊息或目前開啟的檔案已經看得出資料夾路徑，直接用，不用重複問。
- 確認使用者的筆記重點。使用者可能已經在對話中打字講了每張圖（或每個段落）在講什麼，也可能還沒講——如果對話裡已經有筆記內容就直接用；如果完全沒提，才問使用者「想請你講一下這幾張圖大致是在講什麼、重點是什麼」。

## 3. 呼叫 sub-screenshot-rename 改檔名

用 Skill tool 呼叫 `sub-screenshot-rename`，並在呼叫時附上已經確認好的資料夾路徑與筆記重點，讓它不用再問使用者一次。等它回報改名結果（原檔名 → 新檔名對照、最終資料夾裡的檔名清單）。

## 4. 呼叫 sub-screenshot-writer 寫筆記

用 Skill tool 呼叫 `sub-screenshot-writer`，附上同一份資料夾路徑、筆記重點，以及上一步拿到的最終檔名清單，讓它不用再問使用者一次、也不用重新檢查檔名格式。等它回報產出的筆記檔案路徑。

## 5. 回報

用幾句話總結整個流程：處理了幾張截圖、檔名怎麼改的（簡短列 before → after 或說「見上方對照表」）、產出的筆記檔案路徑在哪。不用整篇貼出筆記內容。
